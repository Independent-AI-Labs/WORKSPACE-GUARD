## 1. Architecture Overview

```
  Layer 1: auditd        Layer 2: AIDE/FIM       Layer 3: drift checker
  (kernel-level)        (file integrity)        (baseline diff)
        |                     |                       |
        v                     v                       v
  /var/log/audit/       /var/lib/aide/          res/drift-report.yaml
  audit.log            aide.db                  (stdout summary)
        |                     |                       |
        +---------------------+-----------------------+
                              |
                              v
                    scripts/suid-drift-check
                    (exit non-zero on CRITICAL)
```

Three independent layers monitor the host for tamper. Each layer catches a
different class of attack:

- **auditd**: catches the syscall at the moment it happens (setxattr,
  setcap, open on `.real`, execve of `.real`, mount, open_by_handle_at,
  socket(AF_ALG)).
- **AIDE/FIM**: catches file modifications that have already landed on disk
  (mode change, content change, hash mismatch).
- **Drift checker**: catches structural changes (new SUID binary, new file
  capability, removed immutable flag) by comparing the live surface against
  the committed baseline.

## 2. auditd Rules

### 2.1 Rules file (config/auditd/99-workspace-guard.rules)

```
# WORKSPACE-GUARD auditd rules
# Installed to /etc/audit/rules.d/99-workspace-guard.rules
# Activated with: augenrules --load

# --- Immutable flag / xattr changes ---
# Catches chattr +i / -i and setcap/removexattr on any file
-a always,exit -F arch=b64 -S setxattr -S removexattr -k workspace-guard
-a always,exit -F arch=b32 -S setxattr -S removexattr -k workspace-guard

# --- File capability changes ---
# Catches setcap on any binary
-a always,exit -F arch=b64 -S capset -k workspace-guard
-a always,exit -F arch=b32 -S capset -k workspace-guard

# --- Guard binary tamper (write to *.real files) ---
# The *.real files are mode 0700 root:root, so any write attempt is
# suspicious. We watch the directories where guards live.
-a always,exit -F dir=/usr/bin -F perm=wa -k workspace-guard-real
-a always,exit -F dir=/usr/sbin -F perm=wa -k workspace-guard-real

# --- Direct execve of *.real files (bypass attempt) ---
# A user who tries to run /usr/bin/sudo.real directly is bypassing the
# guard. We log execve on common .real paths.
# (auditd cannot match on filename suffix, so we add per-binary rules
# generated by scripts/sync-gtfobins from the baseline.)
# Example generated rules:
# -a always,exit -F path=/usr/bin/sudo.real -F perm=x -k workspace-guard-bypass
# -a always,exit -F path=/usr/bin/pkexec.real -F perm=x -k workspace-guard-bypass
# ... (one per contained binary)

# --- Copy Fail vector (socket AF_ALG) ---
# If seccomp is not in place, AF_ALG socket creation is the Copy Fail
# (CVE-2026-31431) attack primitive. We log it.
-a always,exit -F arch=b64 -S socket -F a0=38 -k workspace-guard-afalg
-a always,exit -F arch=b32 -S socket -F a0=38 -k workspace-guard-afalg

# --- Mount syscalls (overlayfs LPE vector) ---
-a always,exit -F arch=b64 -S mount -S mount_setattr -S pivot_root -k workspace-guard-mount
-a always,exit -F arch=b32 -S mount -S pivot_root -k workspace-guard-mount

# --- open_by_handle_at (Shocker vector) ---
-a always,exit -F arch=b64 -S open_by_handle_at -k workspace-guard-shocker
-a always,exit -F arch=b32 -S open_by_handle_at -k workspace-guard-shocker

# --- Make audit config immutable ---
# This prevents rules from being modified without a reboot.
# -e 2
# (Disabled by default; enable manually after verifying rules work.)
```

### 2.2 per-binary execve rules

The sync script generates per-binary execve watch rules for every
contained `.real` file. These are appended to the rules file:

```
# Auto-generated by scripts/sync-gtfobins
# Do not edit; regenerate with: make install-auditd
-a always,exit -F path=/usr/bin/sudo.real -F perm=x -k workspace-guard-bypass
-a always,exit -F path=/usr/bin/pkexec.real -F perm=x -k workspace-guard-bypass
...
```

### 2.3 Key-based filtering

All rules use the `workspace-guard` key prefix so the operator can filter:

```bash
ausearch -k workspace-guard --start today
ausearch -k workspace-guard-real --start today
ausearch -k workspace-guard-bypass --start today
ausearch -k workspace-guard-afalg --start today
```

## 3. AIDE / FIM Configuration

### 3.1 If AIDE is installed

The sync script configures AIDE to track:

| Path | What is tracked |
|------|----------------|
| Every `*.real` file | p+i+n+u+g+s+m+c+sha256 (perms, inode, links, owner, group, size, mtime, ctime, sha256) |
| Every guard binary | p+i+n+u+g+s+m+c+sha256 |
| `/etc/passwd` | p+i+n+u+g+s+m+c+sha256 |
| `/etc/shadow` | p+i+n+u+g+s+m+c+sha256 |
| `/etc/group` | p+u+g+s+m+c+sha256 |
| `/etc/gshadow` | p+u+g+s+m+c+sha256 |
| `/etc/sudoers` | p+u+g+s+m+c+sha256 |
| `/etc/sudoers.d/*` | p+u+g+s+m+c+sha256 |
| `/etc/audit/rules.d/*` | p+u+g+s+m+c+sha256 |

The AIDE config is generated at
`config/aide/aide-workspace-guard.conf` and installed to
`/etc/aide/aide.conf.d/99-workspace-guard`.

### 3.2 If AIDE is not installed (sha256sum FIM)

If AIDE is not available, the sync script generates a sha256sum baseline at
`res/fim-baseline.sha256`:

```
<sha256>  /usr/bin/sudo.real
<sha256>  /usr/bin/sudo
<sha256>  /etc/passwd
...
```

`make drift-check` runs `sha256sum -c res/fim-baseline.sha256` and reports
any mismatch.

## 4. Drift Checker (scripts/suid-drift-check)

### 4.1 Inputs

- `res/suid-baseline.yaml` (committed baseline from last `sync-gtfobins`)
- `res/fcap-baseline.yaml` (committed baseline)
- Live `find / -xdev -perm -4000 -type f` output
- Live `getcap -r /` output
- Live `lsattr` on every `*.real` file

### 4.2 Drift classes

| Drift | Class | Why |
|-------|-------|-----|
| New SUID binary on a GTFOBins-listed path | CRITICAL | Potential privilege escalation vector. |
| New SUID binary not on GTFOBins | WARNING | May indicate an apt upgrade or new package. |
| Removed SUID binary | WARNING | May indicate an apt upgrade overwrote a guard. |
| Mode change on a `*.real` file to anything other than 0700 | CRITICAL | Tamper with the real binary. |
| Removed `chattr +i` on a `*.real` file | CRITICAL | The inode is no longer immutable; root can modify it. |
| New file capability on an agent-accessible binary | CRITICAL | Privilege escalation. |
| New file capability on a non-agent path | WARNING | Unexpected. |
| Removed file capability | INFO | Operator or program removed it. |
| sudoers file changed | WARNING | Policy change. |
| `/etc/passwd` shadow hash changed | INFO (expected for password change) | Monitored by AIDE, not drift checker. |

### 4.3 Output

The drift checker outputs to two places:

**stdout** (human-readable):
```
CRITICAL: new SUID binary /usr/local/bin/foo (GTFOBins: true)
WARNING:  removed SUID binary /usr/bin/chage
CRITICAL: immutable flag removed on /usr/bin/sudo.real
INFO:     file capability removed from /usr/bin/ping
```

**res/drift-report.yaml** (machine-readable):
```yaml
drift:
  - path: /usr/local/bin/foo
    type: new-suid
    class: CRITICAL
    gtfobins: true
    timestamp: 2026-07-08T14:32:01Z
  - path: /usr/bin/sudo.real
    type: immutable-flag-removed
    class: CRITICAL
    timestamp: 2026-07-08T14:32:01Z
```

### 4.4 Exit codes

| Code | Meaning |
|------|---------|
| 0 | No CRITICAL drift (warnings/info are allowed) |
| 1 | CRITICAL drift detected |
| 2 | Baseline file missing (run `make sync-gtfobins` first) |

CI pipelines call `make drift-check` as a gate. Exit 1 fails the pipeline.

## 5. Remote Syslog (Optional)

If `config/audit-remote.yaml` is present with a `remote:` key:

```yaml
# config/audit-remote.yaml
remote:
  host: syslog.internal.example.com
  port: 6514
  protocol: tls
  buffer_size: 10000
  buffer_file: /var/log/workspace-audit-buffer
```

The auditd rules are forwarded to the remote syslog server via
`audisp-remote` or `auditd`'s built-in remote plugin. TLS is mandatory.
If the remote is unreachable, events are buffered locally up to
`buffer_size` and flushed when connectivity returns.

This is optional: the program functions fully with local audit logs only.

## 6. Known Residual Risks

| Risk | Layer | Residual | Mitigation |
|------|-------|----------|------------|
| Page-cache corruption (Copy Fail) not on disk | AIDE/FIM | Full miss | auditd catches `socket(AF_ALG)`; seccomp blocks it in sandbox. |
| Guard binary replaced (not on `.real`) | AIDE | Detects after the fact | Guard binary hash in baseline; AIDE flags mismatch. |
| SGID binaries not contained | Drift checker | Monitored but not contained | Future extension. |
| Kernel exploit below auditd | None | Out of scope | Kernel patching is operator responsibility. |
| auditd rules modified | `auditctl -e 2` | Partial | Setting `-e 2` makes rules immutable until reboot. Disabled by default. |
